## Antes de enviar pedidos

Os pedidos do playground são enviados para o **servidor de produção** indicado na referência OpenAPI (`cris.clinicaderesultado.com.br`). Use credenciais e dados de teste com cuidado.

O Mintlify encaminha normalmente os pedidos através de um **proxy**. Se obtiver erros de rede ou CORS ao desativar o proxy nas definições, volte a usar o proxy ou teste com cURL/Postman a partir do seu ambiente.

## Fluxo de autenticação

1. Abra a operação **Login API** (`POST /loginApi`).
2. Envie o corpo JSON com `login` e `password` do utilizador CRIS (sem prefixo Bearer).
3. Copie o **token** devolvido na resposta.
4. Nos restantes endpoints, preencha a autenticação **Bearer** com esse token (ou o formato exato que o CRIS esperar no header `Authorization`, conforme a resposta do login).

O sistema distingue **admin_master** (todas as unidades) e **licenciado_unidade** (só a clínica associada). Muitos filtros exigem `clinica` explicitamente para admin_master.
