---
title: "Apoio — Saída"
description: "Endpoints auxiliares que retornam listas de IDs e valores para uso em filtros e parâmetros de outros endpoints do CRIS."
icon: "wrench"
---

Os endpoints de Apoio retornam listas de referência (IDs e nomes) para serem usados como filtros ou parâmetros nos endpoints principais.

<Note>
  Todos os endpoints de Apoio exigem autenticação via token. Consulte o endpoint [loginApi](/cris/entrada/apoio#login-api).
</Note>

---

## Status Agendamento

<Check>
  **Status: Disponível**
</Check>

Lista as opções de **Status de Agendamento** com seus respectivos IDs.

| Propriedade | Valor |
|---|---|
| Método HTTP | `GET` |
| URL | `https://cris.clinicaderesultado.com.br/rest/v1/statusAgendamento` |

```http
Content-Type: application/json
Authorization: {token_gerado_pelo_loginApi}
```

**Campos retornados:** Agendamento: IDs do Status

## Lista Clínicas

<Check>
  **Status: Disponível**
</Check>

Lista todas as clínicas cadastradas no CRIS com seus respectivos IDs.

- **admin_master:** vê a lista completa
- **Usuário licenciado:** vê apenas o ID da sua própria clínica

| Propriedade | Valor |
|---|---|
| Método HTTP | `GET` |
| URL | `https://cris.clinicaderesultado.com.br/rest/v1/listaClinicas` |

```http
Content-Type: application/json
Authorization: {token_gerado_pelo_loginApi}
```

<ParamField query="pagina_atual" type="integer" required>
  Número da página atual. Ex: `1`
</ParamField>

<ParamField query="por_pagina" type="integer" required>
  Registros por página. Ex: `20`
</ParamField>

**Campos retornados:** Clínica: ID | Clínica: Nome
