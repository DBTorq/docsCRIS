<Note>
  Os campos marcados como **personalizados** foram criados pela CR Sistemas. Os campos **nativos** são padrão do Bitrix24.
</Note>

## Campos de Contato

| Nome | ID no Bitrix | Observação |
|---|---|---|
| Primeiro Nome | `NAME` | Nativo |
| Segundo Nome | `SECOND_NAME` | Nativo |
| Sobrenome | `LAST_NAME` | Nativo |
| Contact ID | `ID` | Nativo |
| Telefone (nativo) | `PHONE` | Retorna todos os telefones preenchidos para o contato |
| Telefone (todos) | `UF_CRM_1673035980` | Campo personalizado — usado para **atualizar telefone via automação** em vez do campo nativo `PHONE` |
| Email (nativo) | `EMAIL` | Retorna todos os e-mails preenchidos para o contato |
| Email (string) | `UF_CRM_1676052930` | Campo personalizado — usado para **atualizar e-mail via automação** em vez do campo nativo `EMAIL` |
| Data de nascimento | `BIRTHDATE` | Nativo |
| Observação | `COMMENTS` | Nativo |

## Campos de Negócio

### Identificação e Dados Básicos

| Nome | ID no Bitrix | Observação |
|---|---|---|
| Nome | `TITLE` | Padrão preenchido com `{ID_deal} \| Nome completo do contato` |
| Contato vinculado | `CONTACT_ID` | Contact ID |
| Responsável | `ASSIGNED_BY_ID` | — |
| Criado por | `CREATED_BY_ID` | — |
| Teste | `UF_CRM_1632755419` | `N` = não \| `Y` = sim |

### Fases e Automação

| Nome | ID no Bitrix | Observação |
|---|---|---|
| Fase (nativo) | `STAGE_ID` | Use **apenas para listar/filtrar** negócios por fase. Ver tabela de IDs abaixo. |
| Fase (personalizado) | `UF_CRM_1669405845` | Use para **mudar o negócio de fase via automação** — preencha com o nome por extenso da fase. Ex: `Perdas` |

### Datas e Controle de Criação

| Nome | ID no Bitrix | Observação |
|---|---|---|
| Criado em (nativo) | `DATE_CREATE` | Somente leitura. Como o servidor Bitrix24 fica na Rússia, pode haver variação de **±6h no fuso horário**. |
| Criado em (deal) `[Relatório]` | `UF_CRM_1648177511` | Campo personalizado com automação de correção de fuso horário. **Recomendado para filtros de listagem.** |
